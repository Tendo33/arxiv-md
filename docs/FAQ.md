# FAQ

本页根据当前代码实现整理，默认以中文说明为主。英文读者建议先看 [README.md](../README.md)。

## 基础使用

### 1. 按钮会出现在什么页面？

按钮会注入到 `https://arxiv.org/abs/*` 这类摘要页，并插在 `Submission history` 下方。项目虽然也给 `pdf/*` 页面加载了 content script，但当前按钮注入逻辑只在摘要页执行。

### 2. `Markdown` 按钮到底会做什么？

这取决于设置页里的默认模式：

- `标准模式`：先尝试 ar5iv，再在失败时回退到 PDF
- `MinerU 模式`：直接把 arXiv PDF URL 提交给 MinerU，后台异步处理

按钮副标题会显示当前模式是 `ar5iv` 还是 `MinerU`。

### 3. `PDF` 按钮和 Markdown 失败后的 PDF 兜底是一回事吗？

不完全一样。

- 页面上的 `PDF` 按钮：直接在内容脚本里抓取原始 PDF 并下载
- 标准模式里的 PDF 兜底：由后台逻辑在 ar5iv 失败后触发

两者最终拿到的都是原始 arXiv PDF，但触发路径不同。

## 转换效果

### 4. Markdown 会保留哪些内容？

当前实现重点保留这些内容：

- LaTeX 公式
- 章节标题
- 引用标记
- 图注、定理、证明等结构块
- 复杂表格的 HTML 结构

其中复杂表格会保留为 HTML 表格，而不是强行转成纯 Markdown 表格，这样能减少合并单元格丢失的问题。

### 5. 图片会一起下载到本地吗？

不会。当前 Markdown 里的图片保留为指向 ar5iv 资源的远程链接，所以阅读图片时仍然需要联网。

### 6. Markdown 文件头里会包含元数据吗？

默认会。设置页里有 `Include metadata` 开关。开启后，导出的 Markdown 顶部会带 YAML frontmatter，包含标题、arXiv ID、来源、作者、年份等字段。

## 模式与任务

### 7. 标准模式是不是“ar5iv -> MinerU -> PDF”的三层自动降级？

不是。当前实现里：

- 标准模式：`ar5iv -> PDF`
- MinerU 模式：单独的手动默认模式，需要 Token

也就是说，标准模式不会在 ar5iv 失败后自动切到 MinerU。

### 8. 为什么我做了标准模式转换，Popup 里却没有记录？

因为 Popup 只显示 MinerU 后台任务，不显示标准模式的 Markdown 成功记录，也不显示普通 PDF 下载历史。

### 9. MinerU 任务结果是什么格式？

是 ZIP 包，不是直接返回 `.md` 文件。任务完成后：

- 后台会触发下载
- Popup 中会保留任务卡片
- 你可以再次点击下载，或者复制结果链接

### 10. 为什么会提示“已有重复任务”？

MinerU 任务会按 arXiv ID 查重。如果同一篇论文已有任务，页面会弹出确认框，让你决定是否强制再创建一个新任务。

## 设置与存储

### 11. 设置页里有哪些真正生效的选项？

当前代码实际使用的主要选项有：

- 转换模式
- MinerU Token
- 是否显示桌面通知
- 是否在页面加载时显示自动转换提示
- 是否在 Markdown 中包含元数据
- 界面语言

### 12. 自动转换会直接开始跑任务吗？

不会。它的真实行为是“页面加载后显示一个自动转换提示框”，由你点击 `Start` 后才正式开始。

### 13. 数据存在哪里？

分成两类：

- `chrome.storage.sync`：设置、语言、统计、MinerU Token
- `chrome.storage.local`：MinerU 任务队列、任务状态、结果链接

如果你登录了浏览器同步账号，`sync` 里的内容可能会跟随浏览器账号同步，而不是只留在单机。

## 故障排查

### 14. 为什么提示 ar5iv 不可用？

这通常表示 ar5iv 还没有该论文的 HTML 版本，或者请求被重定向回摘要页。标准模式下这时会提示你仍可使用 PDF 兜底。

### 15. 为什么会报“扩展已更新或重载，请刷新页面后重试”？

这是内容脚本和后台 service worker 之间的连接失效了，常见于你刚重新构建并重载扩展。刷新当前 arXiv 页面后再试即可。

### 16. MinerU Token 测试失败怎么办？

先确认三件事：

- Token 已正确粘贴
- 网络能访问 `https://mineru.net`
- Token 没有过期或失效

设置页的“测试连接”本质上是拿这个 Token 去请求 MinerU 的任务查询接口，所以网络或鉴权问题都会直接反映在这里。

### 17. PDF 下载失败时先查什么？

优先排查：

- 浏览器下载权限或下载目录限制
- 当前页面是否真的能访问对应的 arXiv PDF
- 是否有浏览器扩展或策略拦截下载
